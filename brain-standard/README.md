# The Brain Standard

The Brain is an implementation-independent Knowledge Operating System designed to preserve, organize, validate, and evolve knowledge across tools, agents, workflows, and storage systems.

## Current Deployment State

Status: Controlled bare usable deployment.

The repository may be used for supervised setup, controlled ingestion testing, and implementation preparation. It is not approved for unrestricted autonomous agent operation.

The current document set includes the constitutional architecture documents, universal knowledge standards, storage standards and layout specification, agent framework standards and specifications, workflow standards, controlled ingestion runbook, implementation profile and setup guide, deployment readiness standard, validation checklist, and operational templates.

Document status, authority, and phase are governed by each document's own metadata block. The README and INDEX are descriptive navigation aids and do not approve, downgrade, reclassify, or supersede any governed document.

## Safe Agent Mode

Agents may inspect, summarize, classify, extract, draft, validate, propose metadata, propose relationships, record provenance, and prepare review outputs.

Agents must not approve, delete, publish, export, migrate, downgrade privacy classification, override validation failures, change Object Identity, or mark records authoritative without human review.

Agents must preserve the distinction between standards, specifications, templates, implementation documents, and operational checklists. Implementation-specific documents may support a deployment, but they do not redefine the implementation-independent standard.

## Known Limitations

- This is a controlled bare usable deployment, not an unrestricted autonomous operating mode.
- Pilot ingestion should use only 5-10 test files unless a human explicitly approves broader scope.
- Validation support includes governed checklist and specification documents; executable validators or implementation-specific automation should be treated as implementation support unless separately governed.
- Implementation documents are reference/support documents and must not be treated as software requirements for all compliant implementations.
- AI-model-specific behavior must not become the standard; agents and tools remain subordinate to governed standards, specifications, workflows, templates, and human review.

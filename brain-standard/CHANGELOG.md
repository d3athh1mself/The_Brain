# Changelog

All notable changes to The Brain Standard repository are recorded here.

This changelog is descriptive. It does not approve, supersede, downgrade, reclassify, or modify any governed document. Document authority, status, phase, and dependencies remain governed by each document's own metadata block.

## [0.1.0] — 2026-06-29

### Added

- Added the controlled bare-usable standard foundation.
- Added constitutional architecture documents for vision, principles, layered architecture, and governance.
- Added Universal Knowledge Standard documents covering Knowledge Objects, Object Identity, metadata, relationships, source references, provenance, lifecycle state, authority level, privacy classification, and validation.
- Added Storage Standard documents for storage architecture and practical storage layout.
- Added Agent Framework documents for controlled agent operating boundaries and implementation-ready agent setup support.
- Added Workflow Engine documents for ingestion, review and approval, maintenance and update, publication, export, and downstream use.
- Added the controlled ingestion runbook for supervised ingestion runs.
- Added the bare-usable deployment readiness standard.
- Added the Knowledge Record metadata field specification.
- Added operational templates for Knowledge Records, Source Material intake, Agent Activity records, and Review records.
- Added the bare-usable validation checklist.
- Added implementation-support documents for Obsidian and Codex as reference implementation support, without making those tools part of the standard.
- Added repository-level `README.md` and `INDEX.md` support files.

### Changed

- Updated repository orientation to reflect controlled bare-usable deployment status.
- Updated repository indexing to map the current governed document set and top-level support files.
- Clarified the project boundary between The Brain Standard, reference implementations, implementation-support documents, agents, workflows, and repository support files.

### Fixed

- Corrected stale repository guidance that previously treated completed files as incomplete.
- Clarified that implementation-support documents may guide practical setup without becoming the standard.
- Clarified that Codex, Obsidian, GitHub, scripts, plugins, file watchers, and any one AI provider must not become the standard.

### Known Limitations

- The repository remains in controlled bare-usable deployment mode.
- The repository is not approved for unrestricted autonomous agent operation.
- Pilot ingestion should remain limited to a small supervised test set.
- Executable validators, automation scripts, and file watchers are not yet governed as standard requirements.
- Publication, export, migration, deletion, privacy downgrade, Object Identity changes, and authority changes still require governed review before use beyond controlled support workflows.
